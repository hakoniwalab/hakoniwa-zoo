# LEGO Digital Twin

## Summary

LEGOなどの教育用ロボットを題材に、実機とシミュレータで同じ制御コードを利用しながら、ロボット開発・制御設計・デジタルツインを学ぶためのユースケースです。

ロボット教育やPoCでは、実機を動かす前にPC上で制御ロジックやセンサ構成を確認したい場面があります。このユースケースでは、ロボットの構造モデルからシミュレーションモデルを生成し、実機と仮想環境を接続する流れを整理します。

## Persona

* ロボット教育教材を作る人
* 実機ロボットとシミュレータを接続したいPoC担当者
* LEGOや教育用ロボットで制御を学びたい学生・開発者

## Problem

実機ロボットだけで開発を進めると、試行錯誤に時間がかかり、故障や破損のリスクもあります。

また、CAD、物理モデル、可視化モデル、制御コードが分離されているため、システム全体の構造を理解しにくくなります。

## Goal

* ロボット構造をシミュレーションモデルへ変換する
* 実機とシミュレータで制御コードを共通化する
* センサ・アクチュエータをPDUとして扱う
* ロボットの構造と制御の関係を学習できる教材にする

## Why Hakoniwa

箱庭では、物理モデル、制御プログラム、通信データを分離しながら接続できます。

そのため、ロボット構造を変更しても制御コードを再利用しやすく、実機とシミュレータを同じアーキテクチャで扱えます。

教育用途だけでなく、企業のPoCやデジタルツイン検証にも応用できます。

## Components

| Component               | Role          |
| ----------------------- | ------------- |
| hakoniwa-mbody-registry | ロボット構造モデル管理   |
| hakoniwa-mujoco-robots  | MuJoCoロボットモデル |
| hakoniwa-core-pro       | PDU通信・時刻同期    |
| hakoniwa-pdu-python     | Python制御プログラム |
| hakoniwa-godot          | 可視化・デジタルツイン表示 |
